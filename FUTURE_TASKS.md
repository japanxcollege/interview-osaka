# 今後の開発タスクリスト

ユーザーフィードバックに基づき、**スマホ・PC両対応（レスポンシブデザイン）** を最優先課題として再構成しました。

## 1. UI/UXのレスポンシブ対応・最適化 (最優先)
**目標**: どのデバイス（スマホ、タブレット、PC）から閲覧しても、崩れがなく快適に操作できる状態にする。

- [ ] **グローバルレイアウトのレスポンシブ化**
  - [ ] **PC表示**: 広い画面を活かしたマルチカラム構成（サイドバー、メイン、プロパティパネルなど）。
  - [ ] **スマホ表示**: 重要コンテンツを中央に配置し、ナビゲーションをハンバーガーメニューやボトムタブに収納。横スクロール発生の防止。
- [ ] **インタビューエディターの最適化**
  - [ ] **Split View (PC) vs Stack View (Mobile)**: PCではエディタとチャットを左右並列、モバイルではタブ切り替え等のUIを採用。
  - [ ] **入力・編集操作**: スマホでのテキスト選択、挿入ボタンの押しやすさ（タッチターゲット44px以上）を確保。
- [ ] **フォント・可視性の調整**
  - [ ] デバイスサイズに応じた適切なフォントサイズと行間の設定。

## 2. アプリケーションの安定化・検証
**目標**: 直近の実装項目（認証、PWA）の信頼性を担保する。

- [ ] **PWAログインの安定性検証**
  - [ ] iPhone/Android実機でのPasskeyログイン、セッション維持の確認。
- [ ] **LINEログインのフォールバック**
  - [ ] 通常ブラウザ環境でLINEログインが引き続き正常に動作するかの回帰テスト。
- [ ] **エラーハンドリング強化**
  - [ ] ネットワークエラーやサーバーエラー時に、ユーザーが自力で復帰（リトライ）できるUIの実装。

## 3. AI機能の強化・可視化
**目標**: AIの挙動を透明化し、ユーザーの安心感と納得感を高める。

- [ ] **AI処理ステータスの詳細表示**
  - [ ] 「構成案を作成中...」「推敲中...」など、具体的な進捗を表示。
- [ ] **完了通知 (Toast)**
  - [ ] 処理完了時に控えめなトースト通知を表示。
- [ ] **生成中アニメーション**
  - [ ] スケルトンスクリーン等を活用し、体感待ち時間を軽減。

## 4. 機能拡張（中期〜長期）
- [ ] **AI生成履歴管理**: 過去のやり取りや生成物を保存・閲覧可能に。
- [ ] **AI設定カスタマイズ**: 生成の頻度やスタイルをユーザー設定可能に。
- [ ] **パフォーマンス最適化**: 長時間録音データのメモリ/通信効率化。
