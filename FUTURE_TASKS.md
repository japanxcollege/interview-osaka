# 今後の開発タスクリスト

このドキュメントでは、インタビューエディターのさらなる進化に向けた今後の開発タスクをまとめています。

## 1. AI機能の強化（短期）
ユーザーがAIの動作状況をより正確に把握し、安心して利用できるための改善を行います。

- [ ] **AI生成プロセスの詳細表示**
  - 「Gemini API呼び出し中...」「原稿構成案を作成中...」など、現在の詳細なステップをUIに表示する。
- [ ] **処理完了のトースト通知**
  - 新しい原稿セクションが追加された際や、新しい質問が提案された際に、画面端にポップアップ通知を表示する。
- [ ] **エラーハンドリングと自動リトライ**
  - APIエラー（レート制限など）が発生した際、ユーザーに分かりやすく通知し、バックグラウンドで自動的にリトライする仕組みを導入する。
- [ ] **生成中アニメーションの改善**
  - 処理中のプレースホルダーやスケルトンスクリーンを導入し、待ち時間の体感ストレスを軽減する。

## 2. ユーザー体験（UX）の向上（中期）
日常的な使い勝手を向上させ、多様なインタビュースタイルに対応するための機能を追加します。

- [ ] **AI処理しきい値のカスタマイズ**
  - ユーザーが「5件ごとに原稿生成」「3件ごとに質問提案」など、自動実行の頻度を設定画面で変更できるようにする。
- [ ] **AI生成履歴の管理**
  - 過去に提案された質問や、生成された原稿の履歴をサイドパネルで確認・再利用できるようにする。
- [ ] **質問提案の多様化**
  - 一度に1つだけでなく、複数の切り口（深掘り、話題転換、事実確認など）から複数の質問を提示する。
- [ ] **エディタ操作の利便性向上**
  - AIが生成したテキストをワンクリックで特定の位置に挿入、または差し替える機能の強化。

## 3. 高度な編集・管理機能（長期）
プロフェッショナルな編集ワークフローを支える高度な機能を導入します。

- [ ] **品質評価・フィードバックループ**
  - AI生成結果に対して「👍/👎」で評価できる機能を付け、その情報をプロンプトの自動調整に活用する。
- [ ] **カスタムプロンプト（スタイル設定）**
  - ユーザー独自の執筆スタイルや、特定のメディア向けのトーン＆マナーをプロンプトとして登録・保存できる機能。
- [ ] **パフォーマンスとスケーラビリティの最適化**
  - 大規模なインタビュー（長時間録音）における、WebSocket通信とメモリ使用量の最適化。
- [ ] **データ基盤の強化**
  - 現在のローカルJSON/PostgreSQLのハイブリッド構成を整理し、ユーザー認証（Auth）と紐付いたセッション管理を完全移行する。
