- [x] **グローバルレイアウトのレスポンシブ化**
  - [x] **PC表示**: 広い画面を活かしたマルチカラム構成。
  - [x] **スマホ表示**: 重要コンテンツを中央に配置し、ナビゲーションをハンバーガーメニューやボトムタブに収納。
- [x] **インタビューエディターの最適化**
  - [x] **Split View (PC) vs Stack View (Mobile)**: モバイルではタブ切り替えUIを採用。
- [ ] **微調整**: 実機確認で見つかった細かな表示崩れの修正。

## 2. アプリケーションの安定化・検証
**目標**: エラー時の挙動を改善し、ユーザーの不安を取り除く。

- [x] **エラーハンドリング強化**
  - [x] ネットワークエラーやWebSocket切断時に、自動再接続や「リトライ」ボタンを表示するUIの実装。
  - [ ] 録音失敗時などのユーザーへのフィードバック改善。

## 3. AI機能の強化・可視化
**目標**: AIの処理状況を透明化し、ユーザーの待ち時間を快適にする。

- [x] **AI処理ステータスの詳細表示**
  - [x] 「構成案を作成中...」「推敲中...」など、具体的な進捗を表示。
- [x] **完了通知 (Toast)**
  - [x] 処理完了時に控えめなトースト通知を表示。
- [x] **生成中アニメーション**
  - [x] スケルトンスクリーン等を活用し、体感待ち時間を軽減（点滅アニメーション採用）。
